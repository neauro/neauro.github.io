The Cutest Macaroon Llamas in the World

Dec 2nd, 2010


In a game center in Osukannon, Nagoya, we found the cutest things in the world. They are Arpakasso*, and even though I have never felt especially fond of llamas, somehow their plush smiles and ambiguous relation to macaroons struck me right in the soul.

ARPAKASSO: OUR HEALING SWEETS ANIMAL.

They were all piled up in such a way that it was super easy (for De) to grab them.



I sound calm now, but at the time we were all jumping up and down and frothing at the mouth from sheer euphoria. HOW CAN CLAW MACHINES DO THIS. HOW CAN THEY CHANGE HAND-SIZED, AMBIGUOUSLY MACAROON-RELATED LLAMAS INTO PURE KAWAII CRACK.

DON’T EVEN UNDERESTIMATE THIS.

ARPAKASSO ARE SO CUTE THEY EVEN MADE ME INTERNET FAMOUS.

(via heatherchan!)

In the end, we managed to get four of them before stopping. Any more, and one of the three of us might have spontaneously exploded from sheer llama-induced happiness.


When I left Japan, I left De 10,000 yen (!$120) so that he could get more and bring them back.


*I initially spelled this as “Arkapasso” for whatever reason. Whoops~

Getting Sandy3D 3.1.2 Working on Flash Builder

Dec 2nd, 2010

Somehow to get Sandy working on my computer took forever, but here’s an example where you can use Sandy 3D/Flash Builder to display a cone. You’ll need to have:

Sandy 3.1.2 [download]
Flash Builder 4
Windows 7
  1. First, download Sandy 3D for ActionScript3 and put it in a place where you can find it.
  2. Create a new Actionscript project in Flash Builder by doing File > New > ActionScript Project. You can name it HelloSandy.
  3. Next is linking Sandy to the project. Right-click on on the project, then select Properties.
  4. Go to ActionScript Build Path on the left-hand sidebar.
  5. Click the Source path tab.
  6. Click Add Folder and find the directory that contains the Sandy version that you downloaded. Add the src/ folder and click OK.
  7. Type in this code:
    package
    {
    import flash.display.Sprite;
    import flash.events.Event;

    import sandy.core.Scene3D;
    import sandy.core.data.*;
    import sandy.core.scenegraph.*;
    import sandy.primitive.*;

    public class HelloSandy extends Sprite
    {
    private var scene:Scene3D;
    private var camera:Camera3D;

    public function HelloSandy()
    {
    // We create the camera
    camera = new Camera3D( 300, 300 );
    camera.x = 0;
    camera.y = 100;
    camera.z = -400;
    camera.lookAt(0,0,0);

    var root:Group = new Group;

    var myCone:Cone = new Cone("theObj1",50, 100);

    root.addChild(myCone);

    scene = new Scene3D( "scene", this, camera, root );

    addEventListener( Event.ENTER_FRAME, enterFrameHandler );
    }

    // The Event.ENTER_FRAME event handler tells the world to render
    private function enterFrameHandler( event : Event ) : void
    {
    scene.render();
    }

    }
    }
    This will create the cone to display.
  8. Now everything should be linked up! Run the program by going to Run > HelloSandy or pressing CTRL+F11.

With the 3.1.2 version of Sandy, I got two warnings in Node.as and VirtualMouse.as in the source code –

1106: Empty statement found where block of code expected. Did you type ‘;’ accidentally?

But if you run the project, the image will still render. You can remove the warnings by removing the “;” at the end of the line in Node.as, and commenting out the weird line in VirtualMouse.as.

Yay~

Note: I sometimes still have trouble loading in the Sandy library, but am pretty sure that it’s mostly related to the fact that I keep the source code in C:\Program Files, and maybe there’s a permissions issue with Adobe Flash Builder trying to access the library from there? Hmm. -shrug-

~*New, Fancy Noobling Blog*~

Nov 26th, 2010

My favorite thing to do is start projects, particularly those that involve huge copious amounts of writing, so it stands to reason that I’ve been wanting for like forever to make a blog in which I could attempt making pro conversation/discussion with myself about things that I want to learn/practice/improve upon in general.

My project threads are as follows:
  • Red Moleskine: For whatever form inspiration strikes in, including abruptly making some observation about the weather, drawing up a database schema for a web app (that I found out has already been done), sketching out shiba inu people, etc. etc.
  • Spotty Daiso Notebook: For taking note of the myriad of programming/software-design related things that I have no idea about, so that I actually look them up later; also, to keep track of tasks for school research projects
  • Opalite Wires: For practicing writing something completely uncalled for every single day
  • Normal Blog: Also for practicing writing something completely uncalled for every single day, but also for purposes of coming back months later to ooze nostalgically over whatever I wrote the previous year and see whether I’m still friends with the people that I talked about
This makes for tons of ideas that are all over the place, and ANYWAY, the point is that, the objective of this blog is: to try and consolidate, into a relatively intelligible way, all the things that I do in all the other threads.

IT WILL MAKE SENSE WHEN THERE’S MORE THINGS.
