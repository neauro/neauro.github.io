Casual Game Binge & Speculation

Sep 5th, 2011

I’ve spent most of the weekend playing hidden object games to try and understand them better. The only one I’d been familiar with before was the Nancy Drew Dossier series, which I’d hesitate to call a “hidden object game” because it feels like it contains more small puzzles than actual “hidden object scenes” where you’re given a I-Spy list of things to spot. Then again, most hidden object games also have small puzzles, so the whole “hidden object” genre in general seems pretty fluid. And I think I ended up playing some games which, like NDD, seem to be general puzzle-solving adventures.

My intention in playing these was to investigate and note what I felt were good and bad design choices, and in general to try and get a grip of what people expect hidden object games to offer. I played all the games for at least an hour. Spoilers may follow.


Drawn: Dark Flight

Drawn: Dark Flight is the sequel to Drawn: Painted Tower, and the premise of the game is to light beacons in order to help a young woman become the Queen of Stonebriar.

(source)

Things I Liked:

  • The art is beautiful. I’m a sucker for dust motes (which seem to be a common theme of most hidden object environments), and the mostly-somber painted style was also very beautiful to look at.
  • I like the subtlety of hints – for instance, the hint You can climb up a paper-thin rope cluing me into using the fact that I had to use my “Sketch of A Rope” object. Haha it’s a dead giveaway, but better than a completely dead giveaway, you know? You still feel kinda smart solving it.
  • The use of the “drawn stuff/paper/sketches” theme and the variety of puzzles yielded by it was also pretty cool.

Things I Didn’t Like:

  • Whenever I try to navigate backwards, I keep accidentally hitting the inventory box at the bottom of the screen. This is because your inventory resides at the bottom of the screen, and also near the bottom of the screen is an invisible hitbox you click in order to navigate backward.

    dang you, invisible hitbox

    This was an annoyance across most games that I played. This is related to Fitts’s Law, which is a model of human movement that basically says, The bigger the button is, the easier it is to point at it. With computers, the edges and corners of a screen are easy to point at, because the mouse cursor remains at the edge or corner of a screen regardless of how much further you move it – thus, I personally found it annoying to want to move “back” and having to restrain myself from going all the way to the bottom of the screen. I suspect (though do not know) that the act of moving backward is more common than looking at your inventory item, so I think that whole lower area of the screen should be reserved for navigation.
  • When you pick up an item, it goes straight to your inventory, and you don’t really have a chance to examine it, much less know what it is that you now have in your inventory. Maybe it’s Zelda that’s made me feel like I should be rewarded/acknowledged every time I come into possession of something, but when you grab something and it spontaneously goes into your pocket before you’re even sure what it is, it feels a little forced.
  • Wait, so who am I again? And why do I care about this girl I don’t know becoming the queen? I know it’s a sequel but it would have been nice to have a little more intrigue. I didn’t play the entire game, though, so maybe there’s more interestingness in the plot later.



Shiver: Vanishing Hitchhiker

You pick up a hitch hiker. As you drop her off in a small town, you realize she left something behind. It’s a teddy bear…from your childhood! This hidden object game has a horror/scary flavor to it.

(source)

Things I Liked:

  • The hidden object scenes were slightly different – rather than just searching the scene, to find some objects, you had to manipulate the environment in some way. For example, some objects would be hidden in a box you’d have to open, or you might have to put together a photograph’s torn edges in order to really “find” the photograph.
  • I liked the use of the flashlight to illuminate only some parts of a hidden object scene as you were searching, and also as a method of exploring the environment; at one point in the basement area my flashlight fell onto a bright-eyed rat and I jumped in my seat, haha. Limiting a player’s view = good shock/scare value.
  • Speaking of good shock/scare values, this game did a good job with creepy things, for example that one hidden object scene where you had to find a hand, and the hand was a plastic doll’s hand in a meat grinder.
  • The environment would also move on its own sometimes, with shadows in the windows or eyes at the keyhole – cool stuff like that.
  • When I was trying to reconstruct steps in the basement and was failing, the main character himself said, “I have to go from the lowest step.” I like that the character himself said something, rather than the game putting up a dialog for me to follow – it’s a little more natural-feeling. On the other hand, I guess this too can have its pitfalls.
  • The story started becoming really, really intriguing after a while, and I really want to play more to see what’s actually happening in this strange little town!

Things I Didn’t Like:

  • Again, objects that you picked up abruptly went into your inventory without announcement or examination.
  • And again with that “let’s reserve the bottom of the screen for the inventory” thing.
  • Something else this game does that seems common to hidden object games – really abrupt hidden object scenes. Like, you approach a desk, and click it, and without warning there’s a list of objects at the bottom of the screen. Huh? No instruction? Why do I specifically need to find wrench, oil can, screwdriver, wood planar? At then at the end I suddenly get to keep the screwdriver?! A lot of puzzles in hidden object games seem to throw you in without any instruction, but for a hidden object scene it’s a little weird, because you have the list at the bottom that is not strictly a part of or even in its entirety related to the “game world” at all. I wonder players find this treatment of hidden object scenes pretty normal. If they do, that is pretty interesting as an example of unspoken expectations between the players and the developers. It would be like making a shooting game and not teaching the player how to use a gun. Maybe there are games like this now.
  • I was also kind of sad there was so little opportunity to learn about the main character of the story, who (judging by the first scene of the game) was slightly psychic. At the very least he wasn’t utterly freaked out by the appearance of ghosts later in the game, and when I hovered over his camera on the bottom-right side the tooltip said that it could see ghosts. So…alright then. My desire to learn more about the main character may have come about through playing Phoenix Wright, and I wish you were able to “interact” with the main character to learn who he is, really. Phoenix Wright is lovable because of his dryness, which is readily evident by what he thinks about the various things you choose to examine in different scenes of the game. But especially with these hidden object games, you’re given such a richer-looking environment than what’s seen in a DS – why not interact with it a little more?
  • Without knowing more about who the character is, the premise of why he ventured into the village – to seek the girl who may be from his past – feels forced and weak. What are the stakes here? I think that there’s another Shiver game out there, but even if this is a sequel, it should be able to stand up on its own and explain itself.
  • At some point I couldn’t find some object and started clicking all over the screen randomly, and my cursor started spinning around of its own accord moved off the screen. For a second I thought this was creep-factor stuff, but then later realized this was the game’s way of penalizing me for clicking around randomly. In which case, a “Don’t do that or else I’ll make your cursor crazy” warning would have been nice.



Tiger Eye: Curse of the Riddle Box Part 1

A woman with an affinity for metal finds a shapeshifter man in a game that is based off a paranormal romance novel. As a sidenote, this game was developed by the people who developed NDD.

(source)

Things I Liked

  • The hidden object scenes were related to what was going on in the story presently. Have a dream about metal weaponry? Find metal weaponry in a Chinese dirt market! Need to assemble a bike? Find the bike parts in an alleyway!
  • The things that you find in scenes also correlate to puzzles – for example, collecting metal capsules which were then used in a word puzzle.
  • You have more hints than other games, and the number of hints you have is directly correlated with your actions, and if you don’t use hints you are rewarded (by bonus material at the end of the game).
  • If you click around too much, your cursor freezes and the game explains to you what you did that it didn’t like.
  • Good voice acting!
  • And pretty scenes!
  • The puzzles are very varied and fun to solve.
  • Bonus: I love when the thing that I’m searching for is everywhere! For example, one scene where pieces of paper were strewn all over the room. Collecting everything in sight is just a rush. Maybe it’s the mad, doober-collecting player that Zynga has bred in me.
  • Super Bonus: A female protagonist who is like, “I can’t wait for the handyman, I’ll just fix the bathroom myself.”

Things I Didn’t Like

  • The object you had to find were pretty small, even in full-screen mode.
  • Each puzzle that you had to complete had instructions telling you what to do on the bottom. A lot of other hidden object games just throw you in. The puzzles are simple enough that you can figure out what to do on your own, and if you are given the chance to figure out a puzzle from scratch, I think the sense of accomplishment is greater. At the very least there’s less of a “hand-holding” feeling. If you really need to give instructions, you could give the player a chance to ask for it via a hint.
  • A lot of these puzzles could have really used a “Restart” button. Too often I messed around with pieces I had to assemble somehow, got completely confused, and then just wanted to start over fresh, without having to move all the pieces back out of the way myself.

    okay…I just want to start over

  • I also wish there had been an ability to exit a puzzle I was stuck at to take a break and check out another puzzle for a while.
  • Also, though there’s a row of puzzle hints you can use at the top of the puzzle (visible in the above screenshot), they were unresponsive to clicking. And though the number of hints I had was still visible in its usual place in the bottom-right, I couldn’t use them. Confusing.



Mystery Legends: The Phantom of the Opera

It’s based off Phantom of the Opera.

(source)

Things I Liked

  • Again, gorgeous art. DUST MOTES. YES. There were also small moving elements I liked, like little spiders everywhere. But again, I wish I’d been able to get a rise out of the main character by clicking them. Is she afraid of spiders? Is she not? I’ll never know.
  • Good tutorial for how to navigate, and I also like that navigational arrows are labeled with where you’ll go should you decide to click. It’s especially useful for this game because there are so many places. The map is also helpful in marking the rooms that have something of note in them.
  • When you find things, they vanish, and sometimes what’s left behind is creepy! Like this one mannequin had a helmet, and when you found the helmet it revealed this rather disturbed-looking head. That is a really good use of that mechanic.
  • The inventory at the bottom of the screen is hidden a little better.
  • I like the mysterious hints that you pick up, i.e. the Green Faerie Absinthe Recipe. Pretty fantastical. Didn’t play far enough to see whether this really played into the story well (other than it being a recipe for something you need to collect items to make).
  • Bonus: There’s a reference to Wicked!

    can you find it?!


Things I Didn’t Like

  • I would have wanted more opportunity to examine things, and more narration to help me learn more about the protagonist, etc.
  • There were so many…places, haha.  Maybe I’ve become too casual for my own good.



Mystery Case Files: Ravenhearst

The Queen of England has enlisted your help in investigating a haunted manor. You uncover what mysterious and terrible things happened therein by collecting the pages of a woman’s diary.

(source)

Things I Liked

  • To actually collect the diary entries, you have to piece them together, which turns out to be unexpectedly challenging. o__o

    the answer is so close, yet so far

  • I liked the whispers in the background – occasionally you’d hear a Help me or This place is evil! whispered alongside the background music. “This place is evil” seems a little corny but I still appreciated it, haha. Audio mechanics, though, remind me of accessibility – a game that relied more heavily on sound effects as part of the plot would be inaccessible to the deaf. It also makes me think of how sound does not matter with many Flash games, since people play them at work with the sound turned down. How often do people play hidden object games with the sound up? You could potentially play games like these in small intervals on work breaks.
  • Stuff in the background changes subtly – for example, clicking on a painting of a woman may change her face very subtly into a skull. Once I clicked on a card and it tore in half, revealing a scrawled red HELP. This is what I mean when I say I like interacting with the environment more. :D Plus, this is how you can contribute and really strengthen your theme.
  • This game has a pretty cool type of puzzle where you come up against a door which is locked with a completely bizarre-looking mechanism. No instructions – just figure it out. “Click everything!”

    just, GO!

  • Clicking stuff at random reveals a help dialog!

Things I Didn’t Like

  • Clicking stuff at random, after the help dialog shows up, gives you a dialog that you shouldn’t do that, and then punishes you by removing around a minute of the time you’re given to solve a “level.” What! I can’t just have a warning at first?
  • When you click on something in a hidden object scene that is not on your I-Spy list, there’s this…really abrasive noise. Which I guess deters you from clicking at random, but…man. That noise is unpleasant.
  • There were small wording issues that I didn’t like…when my list told me to find a “squash” it really meant find a “kabocha,” and when it asked me to find a “shovel” it really wanted me to find a garden spade looking thing. (Then, later, it asked me to find a “spade” which was an object that looked exactly like a “shovel”!) It’s a pretty small issue but it brings to mind the problem with choosing hidden objects to be things that everyone will be able to figure out. I guess if you’re downloading this game from Big Fish it can be pretty assumed you have the internet to Google Image wood planar, so perhaps this is a trivial issue.
  • On a related note, one of the puzzles required me to play music on a piano keyboard, which would have stopped me in my tracks if I didn’t know how to read music or play the piano. One thing I like about the Nancy Drew Adventure games is that if you will need some sort of knowledge to solve a puzzle, the game will provide it for you, for example in a book somewhere. Then again, NDA is a game that could be purchased and installed from a store (i.e. no internet required), whereas these games are downloaded, so, maybe it’s a trivial issue again.
  • The phrase “Hidden Clues Remaining” at the bottom-right – I thought this meant the number of hints I had left overall, but it was actually the number of hidden things I had to find before I could unlock a diary page to piece together. I don’t remember the game explaining that, though.

Still not a casual game pro, but definitely getting more intrigued by them.  It seems like a good area for building strong stories in an industry that otherwise doesn’t seem like it has the time for them.  Though recently I watched the gameplay trailer for Bastion and was pretty impressed and happy at how the game is narrated.

A Recipe for Baked Potatoes

Sep 4th, 2011

Real Talk: This is the only thing I cooked for myself all summer.

dat ingredients


Ingredients

  • 1 Russet Potato
  • 2 broccoli florets
  • 4 white mushrooms
  • some Jarlsberg cheese
  • pinches of rosemary, ground seasalt
Serves 1 (though to be honest, I’m always overly full when I finish, so you could probably split it)


Instructions

  1. Preheat the oven to 400 degrees Fahrenheit.
  2. Wash the potato, stab it all around with a fork, and then stick it in the oven. If you do not stab your potato with a fork or knife, it will explode.
  3. Wash everything and then begin cutting it up. Mushrooms I tend to mince, and for broccoli I tear up the florets by hand. I don’t usually like the stem (despite it actually being sweeter than the florets), but it’s easier for me to eat if I cut it up into small pieces. Might as well use all the pounds you paid for, right? Fun Fact: I can’t find evidence of this now, but my old cooking teacher told me that it’s better to tear broccoli rather than cut it. Fun Fact #2: We’re not using it here, but this reminds me that I read garlic yields more of its nutrients to you if you mince it than if you leave it in chunks.

    i always really liked the cross-sections of mushrooms and broccoli stems

  4. Once the potato has been in for around 30 minutes, throw the broccoli into a pot of water and start bringing it to a boil. Once it starts boiling, turn the heat down low or turn it off completely and let it drain in a colander. This is to prevent the broccoli from becoming over-cooked and leaking all its nutrients into the water besides. (It’s probably better to steam it, but I don’t have a steamer and am always too lazy to look up hacky ways to steam.) The florets can hold a lot of water, so make sure you press out all the moisture either now or as you’re eating so that stuff doesn’t get soggy.
  5. Around this time, I also start frying the mushrooms. Specifically, I do it with Trader Joe’s Spanish Olive Oil, and sprinkle on rosemary and freshly ground seasalt. Why freshly ground seasalt? Because when I searched the cabinets of the place I’m subletting, all I could find was a canister of large seasalt grains and a porcelain mortar and pestle.

    i wonder if it really makes the salt taste better

  6. Once the mushrooms start looking pretty cooked, turn down the heat. It’ll depend on you how cooked you want the mushrooms to be, though I generally like them pretty brown myself. I wouldn’t be frying them for more than 10 minutes or so.
  7. Once the potato has been in the oven for an hour and a half to two hours, take it out. I generally err on the side of waiting longer for the potato, since I love it when they’re soft and mushy on the inside. Plus this makes the skin crispier!

    an armor of crunchy deliciousness


How to Eat

I generally slice the potato in half, put in liberal amounts of minced Jarlsberg, and then hold the potato closed so that it will all melt into gooey deliciousness. Then, eat with the mushrooms and broccoli in whatever proportions you desire.


mmmmmmmmm


Cost

Laziness always brings me to the closest grocery for ingredients, which happens to be Whole Foods. This is how much I spent at my last visit today:
  • White mushrooms, 0.40 lb at 3.99/lb – $1.60
  • Broccoli crowns, 1.02 lb at 2.99/lb – $3.06
  • Russet potato, 1.46 lb at 1.99/lb – $2.91
  • Jarlsberg – $4.00
  • Total: ~$11.57

Everything was organic and thus more expensive, but this sort of stuff always makes me aware that “eating healthy” can really be a class issue. I’m not saying that this is the healthiest meal on the planet, but it’s probably better than a McDonalds dinner, with a trade-off that it’s more expensive money-wise and time-wise. I’m privileged in that I don’t work so hard each day that it’s difficult for me to find time to cook, much less find time to cook for myself and a family. I mean, come on guys. $8 for two potatoes, 5 mushrooms, and 3 broccoli crowns?

[Microaggression] Signing Up for StumbleUpon

Sep 3rd, 2011


Really, StumbleUpon?  You’re going to ask for my gender so that you can find the best sites for me?

Oh, right, because, now that you know I identify as female, you can show me female websites.  Like, ones with makeup.  Or pink.

But…it’s okay!  Because this is what’s best for me.  As a female!

I understand that women do tend to be more into certain things – eyeliner, FarmVille, Edward Cullen, good hair days.  Even I cannot fight against the allure of kittens and arupaka.

But the reason why women tend to be into these things is because we are told, These are the things that you should love, these are the things that are best for you as a woman.


I’m sure that websites have been partitioning people and withholding information inappropriate to their stereotype for a while, but I’ve never seen it admitted so blatantly.  Some time ago I found out that Google shows me the same ads that it would a man 25 to 24 years of age (though now it seems that it just shows me the categories it has me listed under, versus actually giving me a gendered/aged demographic that I fit), but there’s something more insidious about changing the actual websites that I might see, based not on my browsing history, but merely on Are you a boy?  Or a girl?

Anyway, I’ve lived successfully without StumbleUpon thus far, I can probably stand to be without it for a little longer.

Dogs Decoded: Nova

Sep 1st, 2011

I’ve been really into dogs recently (it goes with the whole shiba inu thing, I guess), so last night when I found a new documentary on dogs I jumped on it. I’ve already watched a couple dog-related documentaries but this one still turned out to have some new information. For one, the documentary pointed out that human emotions are not distributed evenly across the face. If you were to split a person’s face in half, and then mirror it to itself to get a complete human face, you’d end up getting two faces that have somewhat different “feelings” expressed in them.

to me, Right seems more confident; Left, benevolent

There’s also a theory that emotions are more faithfully represented in the right side of the face, which is, coincidentally, the side of the face that we as humans tend to look at first when examining a face. They showed a study which showed this, outlining human gazes with lasers and showing that people generally looked at another person’s right eye first, then the left – but to me this kind of seems typical of how I would look at anything first.

for example. (source)

I wonder if this still holds true for humans whose language reads from right to left. Though, I guess regardless of whether the “leftward gaze” does hold true for humans, it does seem to hold true for dogs, who would also look to the right side of a human’s face first.


puppy dog eyes

No other animals interact this way with humans, and no other dogs do this with other dogs. It is almost as if dogs have a special relationship to humans. Whoa.

Another special thing that dogs do: bark. Wolves don’t bark at all, but domesticated dogs have a variety of noises, which their owenrs can often interpret as wanting to play, angry I hate strangers!, etc. In general, people tend to forget on how humans and dogs must have evolved together; in fact, when we think of how humans evolve we tend to look at primates like chimpanzees, but it turns out that chimps, like many other animals, don’t understand the human gesture of pointing.

wat lol give me food

Pointing is a human gesture that is informative in nature, and helping someone to get food by pointing is something that seems to be pretty unique to humans/humans or humans/dogs. Chimps in the wild would apparently never run into a situation where they would point at a banana to indicate its presence to another chimp, though wow, I would be really surprised if this were true. Still, dogs’ ability to read humans is so good that they not only follow pointing, they can follow human gazes. Human eyes, by the way, are pretty easily followed, what with the white sclera and dark iris, and it may have evolved to be this way because it facilitated better communication.


o___o

As for the evolution of dogs, genetic testing seems to prove that they undeniably came from wolves, and not any other dog-like creature like coyotes or foxes. Dogs and wolves can even interbreed; but wolf cubs raised as closely with humans as puppies still don’t grow up domesticated. They don’t react to the pointing thing, they aren’t interested in hanging out for lols, etc. So the domestication of dogs is definitely nature versus nuture. In fact, some people in Russia made up an experiment in which they bred domestication into a species of silver fox, by allowing breeding between foxes only if they were not vicious toward the handlers. Within six generations, researchers had some really adorable, tame foxes.


I can hear Tumblr crying out


Though it turned out, that, the more domesticated silver foxes became…


unnff so fluffy

…the more their physical appearance changed. For example, their their coloring became lighter…




…their ears started to maintain floppiness, and their tails started to curl.




In fact, they started looking like DOGS.

photo by Jason Hargrove, used under CC

Meaning basically that, if you select for tameness in a wild kinda-dog-like creature, you’ll end up selecting for physically dog-like qualities.

In other news, I got a job as a game designer!  (I guess I didn’t do as bad as I thought.) I am excited to go back home and stop programming (on others’ whims) for a while, though one semi-sad thing I must accept is that I will probably have to wait a while to have a stable living situation and stable amount of money before getting the shiba inu that’s been keeping me going all summer.  It’ll just be too hard to manage rent, tuition, and a dog, at least for a while.

Monterey

This past weekend my boyfriend came down to visit. After much ado, we managed to obtain a rental car. Who knew it was so difficult to rent cars when you’re younger than 25 and don’t have a credit card? Age and the ability to spend money you don’t have presupposes responsibility and maturity, I guess.

Though, I’m fine paying an extra $25 for the privilege of being young and having a relatively smaller amount of responsibilities. :D

derpaka & shibaru on the dashboard

starting out

aaaand some 20 miles later

What’s visible from the highways of California always impresses me on a level that I can’t quite explain. I love how the environment seems to change from mile to mile, how one state can be all at once chilly and arid, all farms and then all dunes, here grass and there ice plants as far as the eye can see.

in retrospect, why didn’t we stop here?!

one of the cars outside this place actually had WA plates

spotted jellyfish

Wading Through the Amazon Careers Website

Aug 23rd, 2011

The other day I tried applying to a job through the Amazon Careers website, and the experience was pretty interesting.

The normal Amazon Careers website has a frame to the right which is the container for search results and job details.

aww yeah.  so much opportunity for employment


the first listing

Unfortunately for me at the time, I was given a direct link to an internship which filled up the entire page.

does the font need to be that small?

I think the link itself is deprecated because I couldn’t find the job by searching through Amazon Careers, but I didn’t know this at the time. And the last time I tried to apply for a job to Amazon before this was a couple months ago, so I already had a profile, albeit completely unrelated to the position that I was applying for now. Still, I pressed “Apply Now,” thinking that I could change that information before submitting.

But the next page that showed up said, merely,

wait…what about the first three steps?!

“Step 4 of 4” could only mean “This is the last step before submitting your application,” so I applaud the transparency of it.  Still, for some reason I went and clicked through anyway, thinking that I’d still be able to change what irrelevant or old information that I had on file. I didn’t. Ack!

So I searched in haste for “My Account” to put in my application information. When I eventually found it It looks like this:

unexpectedly sparse

The text is, to me, a little small for a page that has nothing else on it. I guess it’s easily remedied by just increasing the font size.  (Also, is that a dangling list element?)

The “Your Profile” page is similar. This is where you fill in all the information that will be used to apply to whatever job at Amazon that you so desire.

Adding a New Model to a Database in Rails

Aug 16th, 2011

For whatever reason, adding a model to a database in Rails took me forever to understand. Thanks to Kelly, I now have model-making in Rails down to an art.

Want to add a new table or a new column to your Rails application? This is what you do!


1. Create a new .rb file for your model

Let’s say that, just like me, you want to create a new database table in your application to keep track of information about dogs. We (by virtue of the same logic which affords physics problems friction-less vacuums with which to work in) already have a couple tables in our database: one pertaining to users (breeders or owners of dogs), and one pertaining to breeds. We will keep it simple for now; all we want to know about a dog object for now is what its name is, what user it belongs to, and what breed it is.

To add a Dogs table, we first need to create a file which will describe its model. Part of this is describing what the relationship of the new model is to existing models, like User and Breed. From real-world experience, we know that a dog generally has one user/owner, and one breed.

So, create a file in app/models/ called dog.rb, and type in the following:
class Dog < ActiveRecord::Base
  belongs_to :user
  belongs_to :breed
end
Dog inherits Active Record. Active Record’s sole purpose of existence is to help create associations between different objects. This code will later be macro’ed out by Active Record into something more lower-level to create the associations.

The lines that start with belongs_to describe the relationship that this new model has to existing ones. Basically, a dog belongs to one owner/user, and to one breed.

Rails also has another keyword called has_one, which at first might also seem like an okay association to make between dogs and users; after all, a dog has_one user, right? But belongs_to is more correct, because has_one implies more of a one-to-one relationship, where you might be doing something like creating a new user every time you create a new dog. Since users may later obtain more dogs, belongs_to would be more appropriate in the case you want to attach a new dog to an existing user. (Note: belongs_to also makes it so that the Dogs table will have to be responsible for its foreign key with Users, whereas it would be the opposite case for has_one.)


2. Change your existing models to reflect their relationship to the new model

Because a User can have many dogs, and have many different breeds of dogs, User should now look like this: 
class User < ActiveRecord::Base
  acts_as_authentic
  has_many :dogs
  has_many :breeds, :through => :dogs
end
And because many dogs can be of a certain breed, and breeds can be owned by multiple owners, Breeds will look the same – 
class Breed < ActiveRecord::Base
  has_many :dogs
  has_many :users, :through => :dogs
end


3. Generate a new migration

Now you have to create a migration which, when run, will create the relevant tables in a structured manner. You could also be hipster about it and edit SQL on your database by hand, but then you’d would have to tell the other people you’re working with of what you’ve done if they have local databases, or else run the same SQL on production databases. With migrations, you can make changes modular, and even roll back changes easily.

Run rails g migration create_dogs_table. This will create a file in db/migrate which will look something like this:
20110807213821_create_dogs_table.rb
The numbers on the left are a timestamp, which is needed by Active Record so it will know what migrations that you haven’t yet run, so that it can run them for you.



4. Edit the migration file to describe what tables or columns should be added to your database.

Open your migration file and type in these things: 
class CreateDogTable < ActiveRecord::Migration
  def self.up
  end

  def self.down
  end
end
self.up is executed when a database is updated; self.down is executed when a database is rolled back due to errors. So, in self.up you want to create your new tables and columns; in self.down you want to define code to remove those tables and columns. With dog objects, all we said we cared about earlier was the name, so let’s just make a table which will (automatically) contain a unique ID, a timestamp, and a string for each dog object. 
class CreateDog < ActiveRecord::Migration
  def self.up
    create_table :dogs do | d |
      d.timestamps
      d.string :name
    end
  end

  def self.down
    drop_table :dogs
  end
end


5. Run the migration to create your table

Run the migration by typing rake db:migrate. Remember how the migration that we generated had a timestamp? When you run rake db:migrate, Active Record can use those to figure out all the migrations that might have been added to your code base, which have not yet been run on a certain database (such as the one pertaining to production). Also, because the code to create and drop data from a database is so high-level, it doesn’t really matter whether your actual code is in sqlite3 or in MySQL. You can even use both depending on whether you’re in development or production stages.

Hurray! Now you have new tables and columns in your database!
