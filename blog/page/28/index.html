Developing Protocol Buffers

Jan 4th, 2011

Again taken from t3h googs, who developed protocol buffers in the first place.


What are protocol buffers?
Protocol buffers are a “flexible, efficient, automated mechanism for serializing structured data” – like XML but smaller, faster, simpler. In this case, I assume that “serializing data” refers to converting some data structure into an object that can be stored and then unpacked again into another computer environment, i.e. storing a list into a file or something so it can be sent over a network and then opened up again and read by someone else. So, a protocol buffer provides a way for data to be moved across multiple computers, which explains why it’s relevant to distributed systems.


How do protocol buffers work?
You have data, and you want to serialize it so that you can trasmit it to another network or computer in a message. To specify how you want to serialize it, you have to define protocol buffer message types in .proto files. Each protocol buffer message is a small, logical record of information with a series of name-value pairs. A .proto file which defines a message containing information about a Pokemon would be like this:

message Pokemon {
    required string species = 1;
    required int32 id = 2;
    optional string trainer = 3;
    optional string nickname = 4;

    enum Type1 {
        FIRE = 0;
        WATER = 1;
        GRASS = 2;
    }
}

So in this case, I’ve got a message that serializes data about a Pokemon. The message contains information about a Pokemon, its trainer, its id, its nickname, and its type. For a message to be a well-formed Pokemon message, it must contain exactly one of species and id, but can have zero or exactly one of trainer or nickname. The enum means that the Pokemon will have only one of a predefined list of values – that is, only one type out of the three given types.

Once the messages have been defined, they’re run against whatever protocol buffer compiler you have for your application’s language, which will generate data access classes. These will provide a way to access each field defined in the message. So, if you were using C++, the data access class created would be called Pokemon, and you could then use this Pokemon class to populate, serialize, and retrieve Pokemon protocol buffer messages. Then you send out the message like,

Pokemon pokemon;
pokemon.set_species("Charmander");
pokemon.set_id(4);
pokemon.set_trainer("Red");
fstream output("file", ios::out | ios::binary);
pokemon.SerializeToOsteam(&output);

and read it like,

fstream input("file", ios::in | ios:: binary);
Pokemon pokemon;
pokemon.ParseFromIsteam(&input);
cout << "Pokemon: " << pokemon.species() << endl;
cout << "Trainer: " << pokemon.trainer() << endl;

What are the advantages of protocol buffers?
  • You can add new fields to message formats without breaking backwards-compatibility; old binaries will just ignore the new field. So, extending the protocol is simple, and you can do it without having to worry about breaking existing code.
  • It’s simpler, smaller, faster, and less ambiguous than XML. Plus, it generates the data access classes, which are easier to use in programs. I imagine that XML takes so much space and is harder to parse because you have to close tags. Also, to manipulate XML you’d have to do things like finding a node “species” and then accessing its inner text, instead of just retrieving “species” immediately.

Problem: How Do I Use Flex Components to Make UI for an ActionScript Project?

Dec 30th, 2010

I’ve got an ActionScript project which already compiles into a SWF file of 600x400. So how do I add UI (buttons, etc.) to this? i.e. maybe import all the existing AS code into a Flex Project so I can use the fancy UI components?

I near-immediately discovered the SWFLoader, which had a relatively promising name. It’s a control that apparently “lets you load one application into another application.” You add the SWFLoader component, set its Source, and Flex will import the SWF file and run it. SWFLoader can also load different types of images (jpg, png, etc.) but I think that the Image component is supposed to be better suited for that.

Concerning design: an SWF that is loaded with an SWFLoader could be either a sub-application, or a module. The difference is that a main application would load a sub-application, instead of embedding their functionality, which would mean a smaller initial download size and shorter load time for the main application. This also means that you have better encapsulation of the functionality.

In an application that is modular, the main application would load each form as a module using the ModuleManager, which maps URLs to modules, and dispatches events to communicate whether a module is in a loaded state or a not-yet-loaded state.

Making a Flex application work with an embedded SWF
To use the variables and functions in a SWF that’s loaded in your application with a SWFLoader, you need to make sure of these things:
  • import mx.managers.SystemManager
  • Make an instance of a SystemManager, i.e. something like public var loadedSM:SystemManager
  • Make sure the SystemManger instance gets binded to the properties of the embedded SWF with an intializer function that contains a line like loadedSM = SystemManager(SWFLoaderID.content)
  • Make sure that the <mx:SWFLoader> contains a complete property that will call the function that initializes the SystemManager

That being said, here’s a sample for a simple “application” (such a big word for such a silly thing), where clicking on the button changes the text in the Label component:

So, to reference the functions or variables of an embedded SWF, you’ll want to preface variable/function names with loadedSM.application["variable/function name"].

So I guess there’s a much easier way to interact with a SWF inside a SWFLoader, and it goes like this:

  • Interact with it by using loaderID.content

So, to call a function, you could write something like loaderID.content["function"](args).

In Which I Am Confused About Flash, Flex, ActionScript, Etc.

Dec 30th, 2010

So the reason why I was doing all that stuff earlier was because I have to work on adding UI to an ActionScript 3.0 project in Flash Builder.

But apparently ActionScript projects are made using the Flash API, whereas Flex projects are used with the Flex framework. This means that you can’t use Flex UI components with ActionScript projects.

Shoot.

So I wonder how can you have UI element like buttons and such in an ActionScript project? Is it impossible? Is it possible to import the ActionScript code into a Flex project? The issue is that the program I have presently compiles into a SWF of such-and-such given height, and I don’t know how that SWF gets defined so you would be able to arrange it around. For instance, in a Flex project you could use the Design tab to fit stuff together, but I assume that it wouldn’t know the size of a 600x400 coded something until it’s compiled, and wouldn’t know where to place it either.


Edit
Okay, so here’s something else I found somewhere while searching:

There is not support for using the Flex Framework in an AS only project. While it is theoretically possible, you would have to manually bootstrap a lot of the application initialization code that Flex handles (something which would be rather complex).

If you want to play with the framework, but don’t want to layout your app with mxml, then the easier way is to create a simple mxml file that contains an application tag, and a script tag that then points to your code.

This way, the application tag will take care of all of the initialization, and you can then leverage the framework via ActionScript.

SO I GUESS IT’S BETTER TO AVOID USING FLEX COMPONENTS IN AN ACTIONSCRIPT-ONLY PROJECT.

Which kind of begs the question, what’s the difference between Flex and Flash anyway? I mean, I guess I’ve kind of been running around in general because I still don’t quite get the difference between Flex and Flash and Flash Player and MXML and ActionScript and what, even Adobe Air.

So! Let’s have definitions:

Application/Application software: software designed to help a user perform task(s), i.e. writing papers, drawing stuff, listening to music.

Web application: an application that’s accessed over a network like The Internets. It can also refer to an application that’s coded in a browser-supported language, like something that’s a mix of JavaScript and HTML. These are good because web applications can be updated and maintained without distributing and installing software. Examples of web apps: e-mail, online retail sales, wikis.

Desktop application: I assume that an application is a desktop application unless it’s explicitly stated to be a web application? So like, a desktop application for listening to music would be iTunes, but a web application for listening to music would be…like…Pandora.

Software framework is an application skeleton where the application has already defined parts that are the most important – i.e. a software framework for a bank would already have withdraw/deposit methods coded up. They’re a special case of software libraries in that they’re re-usable and have other special features, i.e. inversion of control, which is when the framework dictates the flow of control, and not the caller. Frameworks can also be extended/added to, but not modified.

Rich Internet Application (RIA): “A web application that has many of the characteristics of desktop applications,” which are delivered by way of a browser or browser plug-in, of a sandbox, or a virtual machine. A user would have to install a software framework before using the RIA, and the framework would download, update, verify, and execute the RIA. The three most common platforms for RIAs are Adobe Flash, Java, and Microsoft Silverlight.

Runtime environment: software that supports the execution of computer programs. So, you can’t run a Java program if you don’t have a Java Runetime Enfironment (JRE).

Flash Player: a runtime environment that runs Flash-platform content and applications. Flash Player is used for viewing animations and movies using a web browser. It runs SWF files, which can be embedded in websites. It was originally meant to display 2D vector animations (the likes of which you can build in Adobe Flash), but since then has been also used for making RIAs, and streaming video and audio. And games, and GUI.

Adobe Integrated Runtime (AIR): a runtime environment for building RIAs with Adobe Flash, Adobe Flex, HTML, or Ajax, which can be deployed as desktop applications. So, for a user to use an application executed in Adobe AIR, Adobe AIR must be installed onto the user’s computer at operating-system level. (On a side-note, I noticed Adobe AIR being installed on my computer while installing Rosetta Stone…I guess that’s why.)

Flash: Refers to Adobe Flash (the program), and the Flash Player. Adobe Flash makes flash files, which Flash Player runs.

Adobe Flex: an SDK released so as to develop/deploy RIAs based on Adobe Flash. Flex applications can be written using Adobe Flash Builder (which you have to give moneys for), or just by using a text editor/Eclipse and a Flex compiler. Flex applications are made using MXML and ActionScript. It’s meant for creating RIAs, and provides a lot of useful things, like pretty-looking UI components (buttons, color pickers, grids). Adobe Flex also outputs an SWF file.

ActionScript (3.0): an object-oriented language used for developing websites and software. It outputs into an SWF file, which is then embedded into a website and run by Adobe Flash Player. It was initially meant to control the 2D vector animations made in Adobe Flash.

MXML: used in combination with ActionScript to create RIAs. Flex uses MXML for the layout of the application, and ActionScript for the programming logic. MXML is eventually compiled into ActionScript. Adobe Flash Builder can compile MXML into a SWF file.

SWF file: the output of compiling ActionScript. SWF files can be used by Adobe Flash Player (for the web) and Adobe AIR (for the desktop).


I think the reason for all of my confusion was that “Flash” is apparently used for both Adobe Flash Player, the runtime, as well as Adobe Flash, the authoring environment, as well as like, “Flash games,” which kind of makes you think of some game that’s made “in Flash,” when really it’s written in ActionScript, and compiled to an SWF that is run by Flash Player. And when people compare Flash and Flex on the internets, what they’re really comparing is Adobe Flash and Adobe Flash Builder, probably, not a set of two different languages.

Notes From Me Trying to Learn Flex/Flash Builder

Dec 28th, 2010

Though both Kelly and Nandrew made faces when I said I was trying to learn this

In General:
Flex applications are comprised of MXML and ActionScript. MXML/Flex is used for the application interface, and ActionScript is used for the programming logic. Compiling a project yields a SWF file, which is referenced in an HTML page. When someone looks at the page, the SWF file is downloaded and rendered by Flash Player.

When you create a Flex Project in Flash Builder, you’ll automatically get an MXML file with the same name as the new project.

This is the initial code you get in that MXML file:

<?xml version="1.0" encoding="utf-8"?>

 
      (non-visual elements like services, value objects are placed here)


<s:Application> defines the container for the application; it’s the root tag in the XML document.


MXML and UI
MXML defines what components are in the UI – Buttons, CheckBoxes, DataGrids (which show lists), etc. When you look at the MXML file in Flash Builder, two tabs will appear, which will allow you to alter the layout of the application. You can use the Design tab (when you look at your .mxml file) to add buttons, etc. from the Components bar in the bottom-left, or you can write it buttons manually into the Source with tags that look like <s:Button>, <mx:DataGrid>, etc…

buttons and stuff, yeeeeh

Buttons and whatever you make in the Design tab get converted into MXML. So, looking at the Source tab, this code was generated from my lolsy layout above:

<?xml version="1.0" encoding="utf-8"?>


It doesn’t matter what order the components are in within the MXML file because their positions are determined by the x and y attributes. Other styles can be set this way, like colors – you can view all of them by clicking inside a tag and pressing CTRL+space. Alternatively, all the styles can be done by CSS.

You can get even more information about the components by clicking Help > Dynamic Help and clicking on whatever component you want.

MX and Spark Components
If you take a closer look at the tags for the components added, you’ll notice that their namespaces are different – for instance, the Datagrid tag <mx:DataGrid> has a namespace of mx, whereas the button tag <s:Button> has a namespace of s.

This is due to the fact that Flex 4 introduced a new set of components called Spark components, and uses them along with the older components, called MX Components. Some advantages of the Spark components are that they’re easy to custom skin. In general, Flex 4 lets you choose which components you want to use, and you can mix and match, though use Spark components when you can.

I found more about Spark components and general Flex 4 stuff here.


Making the Button Do Something
To make the button do something, you’ll need to take advantage of the fact that buttons emit “click events” whenever someone presses them. Change the <s:Button tag to include a property click="button1_clickHandler(event)". Flash Builder should fill in that function name as a default, and also add brand new code right beneath the <s:Application> root node, which looks like this:


 <![CDATA[
  protected function button1_clickHandler(event:MouseEvent):void
  {
   // TODO Auto-generated method stub
  }
 ]]>


You can make a pop-up show when someone clicks the button by replacing the commented-out code with Alert.show("LLAMAS"). Flash Builder should automatically import mx.controls.Alert before the function definition, so that the code should now look something like 


 <![CDATA[
  import mx.controls.Alert;
  protected function button1_clickHandler(event:MouseEvent):void
  {
   Alert.show("LLAMAS");
  }
 ]]>



Then you can (build and) run and click the button to your heart’s content.

Switching Between Application States
You can switch between application states by setting currentState to something, like currentState = state2. Placing this in the code for the click event of the button like this:

protected function button1_clickHandler(event:MouseEvent):void
 {
  Alert.show("LLAMAS");
  currentState = "State2";
 }

Will make it so that clicking the button labeled “button!!” will bring you to the page with the llama Image. Though there’s no way to get back to the first state, for now…

    currentState = "State2";
   }
  ]]>

Rosetta Stone Installation Error 9123, 9114

Dec 27th, 2010

omg no stop it die die die

I received Rosetta Stone (version 4) a couple of days ago and was pretty excited about it until I actually tried installing it and kept running into inexplicable errors that apparently no one else On The Internet had/could explain. Basically I was able to install the Application CD alright, but immediately met upon this ambiguous 9123 error when trying to install a language.  (This is on Windows Vista, by the way.)

I resigned myself to customer support chat for a couple of hours, and after following instructions, and then fiddling around with their instructions, I finally arrived at a working solution.

  1. Install the Application CD.
  2. Put in the language level CD that you want to install.
  3. Copy all the files in the language level CD to an empty flash drive.
  4. Remove the language level CD from the CD drive, but leave the flash drive in.
  5. At this point I think you could proceed to install a language by clicking Add a Language in Rosetta Stone TOTALe.  TOTALe should be able to automatically find the language files in your flash drive.
  6. For me, just starting to install like this didn’t work, so I had to go to Start and type in msconfig which will bring up System Configuration.
  7. Click Selective startup.
  8. In the Services tab, check Hide all Microsoft services.
  9. Click the button which says Disable all.  All the services should now be unchecked.
  10. Go back through the list of services and find the one called RosettaStoneDaemon and make sure that it is checked.  If it isn’t, you’ll get fatal errors when trying to start up TOTALe again.
  11. Click Apply and restart your computer.
  12. Once your computer has loaded in again, start up TOTALe and try Add a Language with the language files on your connected flash drive.
  13. Rinse and repeat with any other levels that you have.

And it works!  Actually, I haven’t yet finished activating it so I’m not sure if it works completely, but for now at least, it’s installed.

Getting Sandy3D 3.1.2 Working on Flash Builder

Dec 2nd, 2010

Somehow to get Sandy working on my computer took forever, but here’s an example where you can use Sandy 3D/Flash Builder to display a cone. You’ll need to have:

Sandy 3.1.2 [download]
Flash Builder 4
Windows 7
  1. First, download Sandy 3D for ActionScript3 and put it in a place where you can find it.
  2. Create a new Actionscript project in Flash Builder by doing File > New > ActionScript Project. You can name it HelloSandy.
  3. Next is linking Sandy to the project. Right-click on on the project, then select Properties.
  4. Go to ActionScript Build Path on the left-hand sidebar.
  5. Click the Source path tab.
  6. Click Add Folder and find the directory that contains the Sandy version that you downloaded. Add the src/ folder and click OK.
  7. Type in this code:
    package
    {
    import flash.display.Sprite;
    import flash.events.Event;

    import sandy.core.Scene3D;
    import sandy.core.data.*;
    import sandy.core.scenegraph.*;
    import sandy.primitive.*;

    public class HelloSandy extends Sprite
    {
    private var scene:Scene3D;
    private var camera:Camera3D;

    public function HelloSandy()
    {
    // We create the camera
    camera = new Camera3D( 300, 300 );
    camera.x = 0;
    camera.y = 100;
    camera.z = -400;
    camera.lookAt(0,0,0);

    var root:Group = new Group;

    var myCone:Cone = new Cone("theObj1",50, 100);

    root.addChild(myCone);

    scene = new Scene3D( "scene", this, camera, root );

    addEventListener( Event.ENTER_FRAME, enterFrameHandler );
    }

    // The Event.ENTER_FRAME event handler tells the world to render
    private function enterFrameHandler( event : Event ) : void
    {
    scene.render();
    }

    }
    }
    This will create the cone to display.
  8. Now everything should be linked up! Run the program by going to Run > HelloSandy or pressing CTRL+F11.

With the 3.1.2 version of Sandy, I got two warnings in Node.as and VirtualMouse.as in the source code –

1106: Empty statement found where block of code expected. Did you type ‘;’ accidentally?

But if you run the project, the image will still render. You can remove the warnings by removing the “;” at the end of the line in Node.as, and commenting out the weird line in VirtualMouse.as.

Yay~
